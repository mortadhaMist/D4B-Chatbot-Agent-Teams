# Teams Integration: Azure Bot Registration, Env Vars, and Valid Manifest

This guide describes the exact steps to register your Teams bot in Azure, configure the environment variables for `server.js`, and prepare a valid Teams app manifest.

## 1) Azure bot registration

### A) Create the Azure bot
1. Open the Azure Portal: https://portal.azure.com
2. Search for **Bot Channels Registration** and create a new resource.
3. Fill in the required fields:
   - **Resource name**: any name, for example `d4b-chat-bot`
   - **Region**: choose your Azure region
   - **Pricing tier**: F0 is usually fine for development
   - **Messaging endpoint**: temporarily set `https://example.invalid/api/messages`
   - **Microsoft App ID**: auto-generated by Azure
4. Create the bot.

### B) Collect credentials
1. After creation, open the bot resource.
2. Go to **Settings** → **Configuration**.
3. Copy the **Microsoft App ID**.
4. Under **Certificates & secrets**, create a new client secret.
5. Copy the secret value immediately. This is your `MICROSOFT_APP_PASSWORD`.

### C) Update the Azure messaging endpoint
1. If you are testing locally with ngrok, start your bot server first: `npm start`
2. Confirm the app is reachable at `http://localhost:8080` in your browser.
3. Then start ngrok:
   - On Windows: `ngrok.exe http 8080` or `ngrok http 8080`
   - If ngrok is not installed, download it from https://ngrok.com/download and add it to your PATH.
   - If ngrok requires authentication, run: `ngrok config add-authtoken <your-token>`
4. Copy the HTTPS URL from ngrok, for example `https://abcd1234.ngrok.io`
5. In Azure Bot Channels Registration, set the messaging endpoint to:
   ```text
   https://abcd1234.ngrok.io/api/messages
   ```
6. If using a real public host, set the endpoint to:
   ```text
   https://your-domain.com/api/messages
   ```

## 2) Environment variables

Your bot depends on the following env vars in `.env`:

```env
MICROSOFT_APP_ID=your-azure-app-id
MICROSOFT_APP_PASSWORD=your-azure-app-secret
SERVER_URL=https://your-public-domain.com
MISTRAL_API_KEY=your-mistral-key
SKIP_APPROVAL=false
```

### Notes
- `MICROSOFT_APP_ID` must match the Azure App ID from your Bot registration.
- `MICROSOFT_APP_PASSWORD` must be the client secret value created in Azure.
- `SERVER_URL` is optional but recommended if your server is publicly hosted.
- `MISTRAL_API_KEY` is used by the assistant engine in this app.
- Restart the server after editing `.env`.

## 3) Valid Teams manifest

Edit `teams-app/manifest.json` and make these replacements:

1. `id` → unique GUID for your Teams app.
2. `botId` → your Azure `MICROSOFT_APP_ID`.
3. `validDomains` → your public domain.
4. `staticTabs[].contentUrl` → the publicly reachable URL for `public/chat.html`.

### Manifest validation rules

## SignInAudience and tenant scope (important)

- If you registered your Azure app as **Single tenant** ("Accounts in this organizational directory only"), it can only be used inside that Azure AD tenant. Microsoft Teams may route messages through internal proxy tenants, so a single-tenant app can be rejected with an AADSTS650059 error when used from other tenants or via Teams service proxies.

- To allow the bot to operate across tenants (installable or callable from other organizations), change the app registration's **Supported account types** to:
  - "Accounts in any organizational directory (Any Microsoft Entra ID tenant - Multitenant)"

- Steps to change (Azure Portal):
  1. Go to **Azure Active Directory** → **App registrations** and open your app (Application ID).
  2. Select **Authentication** on the left.
  3. Under **Supported account types**, choose **Accounts in any organizational directory (Any Microsoft Entra ID tenant - Multitenant)**.
  4. Save the change.

- If you must keep the app single-tenant, ensure the bot is installed and used only within the same tenant as the app registration, and set `MICROSOFT_APP_TENANT_ID` in your `.env` to that tenant's ID. The server already supports `MICROSOFT_APP_TENANT_ID` via `channelAuthTenant` in `server.js`.

Refer to Microsoft docs for more details: https://go.microsoft.com/fwlink/?linkid=2295419
- `id` must be a valid GUID.
- `botId` must be your Azure App ID.
- `validDomains` must only include domains your app will call.
- `contentUrl` must use HTTPS and match the domain listed in `validDomains`.
- `outline.png` and `color.png` must exist in `teams-app/`.
